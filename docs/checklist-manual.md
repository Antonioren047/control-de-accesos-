# Checklist manual

- [ ] El panel abre en 1440 px, 768 px y 375 px.
- [ ] El menú se colapsa en escritorio y abre como drawer en móvil.
- [ ] Tema claro, oscuro y automático persisten.
- [ ] El instalador rechaza campos vacíos, correo inválido y contraseña débil.
- [ ] Una instalación correcta crea .env, tablas, catálogos, empresa y usuario.
- [ ] Una segunda visita muestra el instalador bloqueado.
- [ ] /api/health responde JSON y no muestra credenciales.
- [ ] /docs muestra Swagger UI.
- [ ] 403, 404 y 500 no muestran trazas en producción.
- [ ] Un vigilante registra un evento normal, importante y crítico sin poder reducir la prioridad predeterminada.
- [ ] Foto y video se capturan desde cámara; no aparece selector de galería y el video se detiene a los 30 segundos.
- [ ] El evento rechaza una evidencia 11 o un total superior a 20 MB.
- [ ] Un supervisor comenta y cancela un evento dentro de su alcance; otro lugar devuelve 403/404.
- [ ] No se pueden abrir dos recorridos ni finalizar uno sin evidencia y observaciones.
- [ ] Un recorrido excedido aparece incompleto y exige comentario para el cierre supervisor.
- [ ] El cierre normal del turno se bloquea hasta registrar la novedad.
- [ ] El vigilante entrante consulta las últimas novedades del mismo punto.
